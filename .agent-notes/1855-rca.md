# 1855 RCA — three-engine ratio=fill canvas divergence

Input: `tests/1855.dot` — 32-node star (2..32 -> 1), `size="10.0624,6.11863"`,
`ratio=fill`, `overlap=false`, `splines=true`. Diverged under circo (469),
twopi (871), osage (1250) at 0.01; recorded firstDiff = whole-canvas
`[graph] _draw_ filled_polygon` ~1-2pt. Prior note called it a "PRISM
near-miss" — REFUTED below.

## Mechanism #1 (SHARED, FIXED) — `size` not rounded to integer points

C stores the `size=` attribute via `getdoubles2ptf` → `POINTS(xf)` where
`POINTS(a_inches) = ROUND(a_inches*72)` (integer points), per
`lib/common/input.c:476` + `lib/common/geom.h:62`. The port's neato-family
drawing parse (`src/layout/neato/set-aspect.ts` `neatoSizePoints`) multiplied
`inches*72` WITHOUT the ROUND, storing float points.

- C size:   `724, 441`         (ROUND(724.4928), ROUND(440.54136))
- Port size:`724.4928, 440.54136`

`_neato_set_aspect` (`lib/neatogen/neatosplines.c:1041`) forms the fill factors
`xf = size.x/bb.x`, `yf = size.y/bb.y`; the ~0.1% size error perturbs the
scale.

### Evidence (C-instrumented `_neato_set_aspect`, probe reverted)
Natural pre-fill bb is BIT-IDENTICAL C vs port:
- circo bb.ur 622.327317,533.651462 (both); twopi 701.936180,574.811023 (both);
  osage 344,236 (both).
Only `size` differed. circo: C xf=1.407793 vs port 1.410213 (ratio 1.00172) —
exactly the observed node-x and canvas-width delta (877.62 vs 876.11). Y matched
exactly because the R_FILL "else" branch leaves `yf=1` (Y unscaled). osage
applies both factors → port hit exactly `size*72`=724.49x440.54, oracle 724x441.

This is the SAME family as the 2026-07-11 `1949` fix (neato-family omitted
`POINTS()`'s ROUND on `nodesep`). Here it is `size`.

### Fix
`neatoSizePoints` now delegates to the already-faithful `parseDrawingSize`
(`src/gvc/viewport.ts`, which rounds via `toPoints = Math.round(in*72)` — for
positive size, half-up == C's half-away-from-zero). Removes a divergent
duplicate; dot and neato paths now round size identically.

### Result
- circo: 469 → **0 (CONFORMANT)**
- twopi: 871 → 842 (residual = mechanism #2)
- osage: 1250 → 110 (residual = mechanism #3)

## Mechanism #2 (twopi residual, SEPARATE — not fixed) — twopi radial layout

twopi node positions diverge ~1-2pt (by-ID). C-instrumented PRE-scale ND_pos
(inches) at `_neato_set_aspect`, node "1": port y=3.998187 vs C y=3.985300
(Δ0.0129in=0.93pt), x identical; nodes 16/32 differ in both axes. The
divergence is in the twopi natural layout (`lib/twopigen` circle positioning),
BEFORE and independent of fill scaling. Natural bb matched to 6 decimals only
because the bb-defining extreme nodes coincide while interior nodes differ.
Distinct pre-existing mechanism; requires a twopi-layout investigation.

## Mechanism #3 (osage residual, SEPARATE — not fixed) — obstacle splines

osage node centers are now BIT-EXACT vs oracle; all 110 residual diffs are edge
`unfilled_bezier` control points. Edges are obstacle-routed 7-point curves
(`B 7`), e.g. 16->1 differs ~10pt per control point. osage routes via
`spline_edges0`→neato spline routing on the (exact) scaled nodes; the residual
is in the neato obstacle-avoidance spline path, not layout. Distinct
pre-existing mechanism.
