Diagnosis

For the minted four the full SVG (geometry + the title fix: ports, &#45; hyphen, compass-replaces-field) matches dot 15.0.0; each carries a TS portReference drift pin at 0.01pt.

For TOP-steering / multi-rank / record-steering the edge path matches dot within 0.5pt (pinned in the SR4/SR7 oracle tests); the golden fails only on svg/@viewBox, @height and the derived graph transform — TS’s drawing bounding box is ~4–5pt larger for loops bulging beyond the node column.

Why excluded (not fixed)

A bbox-extent computation difference (position-bbox / postproc), not edge routing and not the svgBeginEdge title seam. Changing it risks the 115 existing no-port goldens. The edge geometry is already pinned by unit tests; only the golden mint is deferred.
