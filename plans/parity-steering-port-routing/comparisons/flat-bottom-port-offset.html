Diagnosis

Root cause 1 — bottom-tail (:s) is the drawing transform, NOT a routing error. The constant 7.06pt offset is exactly the graph-transform delta: dot emits translate(4, 64.5), graphviz-ts translate(4, 71.56) — difference 7.06. The edge spline is identical in internal (un-transformed) coordinates; only the drawing bbox (viewBox height 68 vs 76) and its derived translate differ, shifting every rendered point by a constant. This is the same bbox-extent divergence as port-golden-bbox (the loop dips below the node baseline; TS reserves ~8pt more bbox below). The flat routing is faithful.

Root cause 2 — adjacent endpoints use a different algorithm (now ported). When A and B are adjacent (no node between them), dot never uses the box channel: make_flat_edge dispatches to make_flat_adj_edges, which clones the two endpoints + edges into a 90°-rotated auxiliary graph (with a heavy ordering edge), lays it out (dot_rank/mincross/position), repositions, routes (dot_splines_ + gvPostprocess), and copies the spline back via transformf. This is now ported AND the magnitude is fixed: graphviz-ts M27,-37 C27,-64.88 81.63,-68.47 95.72,-47.78 vs dot M27,-37.32 C27,-65.2 81.63,-68.79 95.72,-48.1 — a uniform 0.32pt offset on every control point (start-clip renormalization), not a magnitude error. The earlier half-height arc was the same both-ends-side-port corridor bug fixed in An-Bs: the aux edge for make_flat_adj_edges is a both-ends case, so its corridor was ~16pt too narrow until MINW was applied per rank (edge-route-rank.ts). Pinned as a dot oracle at tol 0.5 in splines-flat-oracle.test.ts.

Drawing bbox — the loop is no longer clipped. The flat loop rises above the node rank (apex y≈-68); the copy-back was not growing the graph bb, so the canvas was only 44pt tall and cut the top of the loop off. Ported the missing update_bb_bz(&GD_bb(g), cp) from dotsplines.c:1270 into the flat-adj copy-back: TS now renders height 76 (dot 70). The ~6pt over-reservation is the control-hull bb approximation shared with port-golden-bbox (C refines to the tighter curve); the loop now fits the canvas. A non-clip assertion guards it.

Status

Bottom-tail is the shared drawing-bbox extent (geometry faithful) — same bbox divergence as port-golden-bbox. Adjacent flats now match dot within 0.32pt via the ported make_flat_adj_edges + the per-rank MINW bound fix, pinned as a dot oracle. No 115-golden uses a flat edge, so output is byte-stable.
