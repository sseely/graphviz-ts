# T0.1 ‚ÄĒ Instrument C `make_regular_edge`, capture ground truth

## Context
graphviz-ts is a faithful TS port; the C source is the spec. Parallel/opposing
cross-rank edges are routed by C `make_regular_edge` (`lib/dotgen/dotsplines.c`,
~line 1880+): it builds boxes per edge, routes each through `Pshortestpath` +
`Proutespline`, and applies a per-edge perpendicular offset. The port instead
routes ONE base then x-shifts (see decisions.md ADR-3). We must capture exactly
what C does for the motivating case before changing anything.

## Task
1. Add debug dumps to `~/git/graphviz/lib/dotgen/dotsplines.c` `make_regular_edge`
   (and `Pshortestpath` call site): for each routed edge print the box list
   (`Boxes`/`boxn`), the tail/head **offset ports** (`ED_tail_port`/`ED_head_port`
   .p, post-offset), the resulting Pshortestpath polyline (`pl`), and the final
   bezier point count installed.
2. Rebuild only the dot-layout plugin to `/tmp/gvplugins` (do NOT rebuild
   libgvc): recipe in `[[recover-slack-and-c-harness]]` /
   `.agent-notes/opposing-edge-spline-divergence.md` (build_dot ‚Üí `/tmp/gvplugins`).
3. Render `graphs/ldbxtried.gv` with the instrumented oracle and capture the dump
   for the three `n0 -> n2` edges (n0 inside `cluster0` x[69,410]; n2 ellipse at
   848,-203 outside).
4. Build a **minimal synthetic repro** that reproduces the under-segmentation
   (parallel edges from a node inside a cluster to an outside node spanning ‚Č•2
   ranks) and capture its dump too. Keep it as small as possible.
5. Record everything in `comparisons/c-oracle-dump.md`: per-edge box lists, offset
   ports, polylines, pt-counts; note whether C offsets the **endpoints** before
   routing (the [[opposing-edge-spline-divergence]] hypothesis) vs the boxes.
