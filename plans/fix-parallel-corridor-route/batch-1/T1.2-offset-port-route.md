# T1.2 — Per-edge offset-port corridor routing

## Context
Core fix (ADR-3). `routeParallelEdgeGroup` currently routes ONE base spline from
un-offset node centers (`baseSplineForGroup`) then x-shifts copies. C
`make_regular_edge` routes EACH edge through its own boxes/corridor from per-edge
**offset ports**, so the line clears obstacles (e.g. the cluster corner) instead
of clipping and under-segmenting. Implement the per-edge frame T0.3 specifies.

## Task (refined by T0.3 root-cause)
1. In `routeParallelEdgeGroup` (`splines-route.ts`): for cnt>1, compute each
   edge's perpendicular offset and apply it to the **endpoints/ports BEFORE
   routing** (per [[opposing-edge-spline-divergence]] — "the lever is the endpoint
   ports, not the boxes"), then route each edge through the corridor via the
   regular-edge router — rather than routing one centered base and shifting the
   whole curve.
2. In `baseSplineForGroup`/`routeRegularEdgeFaithful` (`edge-route-faithful.ts`):
   accept the per-edge offset ports and route from them; preserve the existing
   single-edge path byte-for-byte when cnt==1.
3. Mirror C allocation/installation order (first orig edge → leftmost offset;
   back-edge members reversed by the post-group `edgeNormalize`).
4. Make the T1.1 goldens pass; keep the diff minimal and within the write-set.
