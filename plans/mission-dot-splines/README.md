# Mission: dot-splines (DOT-1) — retire the simplified fitter

## Objective

Route **all** regular (non-flat) dot edges through the faithful pathplan path
(`make_regular_edge` + `routesplines`, already ported and working) and **delete
the simplified fitter** (`computeSpline` / `computeSplineMulti` /
`buildRankCorridor` / `straightEdgeSplineWithRank`). The fitter is a
non-faithful shortcut that diverges from dot on edge-condition geometries the C
algorithm handles — re-verification found the outermost edges of wide fan-outs
/ fan-ins collapse to ~0.4pt stubs (effectively invisible) and rankdir=LR spans
drift ~10pt. `src/pathplan/` is complete, so this is **not** a pathplan port.

Full re-verification + mechanism: `.agent-notes/dot-splines-reverification.md`.
Corpus harness: `.probes/dot-splines-corpus.ts`.

## Branch

`feature/dot-splines` off `main` (already created, HEAD = main). Merge back with
a **merge commit** when gates pass.

## Execution model

Run with **opus** (`claude-opus-4-8`, native 1M context). Measure-first: Batch 1
produces a divergence inventory that refines the fix-batches; log refinements to
the decision journal.

## Strategy (AD-1)

Goldens are generated by the C binary, so a *correct* faithful port must
reproduce them byte-for-byte. Therefore: migrate one edge category at a time to
the faithful path, run the 115 goldens after each, and treat any golden that
shifts as a **faithful-path bug to fix** (never regenerate goldens to match TS).
Delete the fitter only once nothing depends on it.

## Quality Gates (run after every task)

```
- command: npx tsc --noEmit
  pass: exit 0
  on_fail: fix_and_rerun
- command: npx vitest run
  pass: exit 0 AND failed == 0 AND passed >= 1800 AND 115 goldens byte-identical
  on_fail: fix_and_rerun
- command: npx lizard <changed files> -C 10 -L 30 -a 5
  pass: no violations (30 lines/fn, CCN 10, 5 params, 500 lines/file)
  on_fail: fix_and_rerun
- command: git diff --name-only HEAD~1
  pass: within the task's declared write-set
  on_fail: stop
```

Baseline at mission start: **1800 passed / 0 failed, 115 goldens byte-identical**
(main, 2026-06-16). Oracle: `~/git/graphviz/build/cmd/dot/dot` with
`GVBINDIR=/tmp/gvplugins`.

## Operational readiness

- **Observability:** N/A — pure layout, no runtime services.
- **Rollback:** Reversible — revert commits; goldens stay byte-identical so no
  data/format migration.
- **Backwards compat:** SVG output changes ONLY for currently-broken cases
  (improvement). The 115 goldens stay byte-identical. New oracle pins are added
  for fixed cases.

## Constraints (stop / push-forward)

**Stop and wait for human input when:**
- Any of the 115 goldens changes byte-for-byte AND faithful output also diverges
  from the dot oracle (real regression) — or matches dot but differs from the
  stored golden (stale golden; do NOT regenerate without confirmation).
- The T1 inventory shows a large delta set (> ~20 goldens shift) — reassess
  scope before grinding.
- A divergence requires changes outside the routing modules (`edge-route*.ts`,
  `splines-route.ts`, `edge-route-poly.ts`, `edge-route-routing.ts`,
  `edge-route-faithful.ts`, `edge-route-chain.ts`) — e.g. mincross/position —
  meaning the gap is positional, not routing.
- The same location is changed 3× without closing the same failing check.

**Push forward with judgment when:**
- A case reaches dot within tol 0.5 — pin it and move on.
- A sub-case cannot reach parity — quarantine with a `comparisons/<case>.html`
  page + journal reference (AD-5), then continue.
- Minor box/index/off-by-one corrections within the routing write-set.
