# T3 — Parity dashboard + divergence triage

## Context

Same mission context as [T2-survey.md](T2-survey.md). T2 produced `parity.json`
(per-input verdicts). T3 turns it into a human-readable **dashboard** and a
**triaged backlog** that names the next missions, then links it as the
comparison page `CLAUDE.md` requires.

## Task

1. `dashboard.ts` — read `parity.json`, write `PARITY.md`:
   - **Summary** at the top (front-loaded): total, and counts per verdict
     (`conformant` / `structural-match` / `diverged` / `errored` / `timeout` /
     `oracle-error`), plus the quarantine totals per reason (from
     `corpus-manifest.json`).
   - **Diverged + errored tables**: one row per input — `id`, `verdict`,
     `maxDelta` or `firstDiffPath` or `errMsg`. Sort worst-first.
   - **Byte-match** list collapsed to a count + ids (no need to enumerate fully).
2. `triage` the non-matching set into **named buckets** by likely root cause
   (group `errored` by error message; group `diverged` by `firstDiffPath`
   shape — e.g. node-coord vs path-`d` vs missing-text vs viewBox). For each
   bucket: count, example inputs, and a one-line hypothesis. This bucket list IS
   the prioritized backlog for follow-on fix missions. Put it in `PARITY.md`
   under "## Divergence backlog (next missions)".
3. Link `PARITY.md` from `plans/port-catalog/README.md` (the §"Verifying
   fidelity" section) as the realized comparison page / parity dashboard, and
   record the headline counts there.

Do NOT fix any divergence (AD-5). Triage = categorize + hypothesize, not fix.

## Acceptance criteria (Given/When/Then)

- **Given** `parity.json`, **when** `dashboard.ts` runs, **then** `PARITY.md`
  shows the verdict summary whose counts sum to `total`, plus a diverged/errored
  table sorted worst-first.
- **Given** the non-matching inputs, **when** triaged, **then** `PARITY.md` lists
  named divergence buckets, each with a count, ≥1 example input, and a one-line
  hypothesis — usable directly as the next-mission backlog.
- **Given** the port-catalog, **when** read after T3, **then** it links
  `test/corpus/PARITY.md` as the parity dashboard and states the headline
  conformant / diverged counts.
- **Given** the dashboard, **when** reviewed, **then** no `src/` file was changed
  (AD-5) — `git diff --name-only` shows only `test/corpus/*` and the catalog doc.
