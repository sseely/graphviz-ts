- **Corpus path:** `1308_1.dot` · **engine:** `dot`
- **firstDiffPath:** `svg/g[1]/g[1]/text[1]/text()[1]`
- **Port:** oracle unavailable (native dot error) — parser rejects: `Expected … but "?" found`

- **Root-cause group:** G4 — charset=latin1 / ISO-8859 input
- **Why deep:** File contains ISO-8859 bytes 0xDA (×6) in identifier names; C treats bytes ≥0x80
  as NAME characters, but the JS harness reads the file as UTF-8, producing U+FFFD replacement
  chars that the PEG grammar rejects. Requires encoding-detection or latin1 re-decode at the
  harness layer (ADR-5).
- **Follow-on bucket:** `charset-encoding`
