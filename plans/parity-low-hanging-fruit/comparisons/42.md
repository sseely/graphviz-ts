# Deep case: 42

- **Corpus path:** `42.dot` · **engine:** `dot`
- **firstDiffPath:** `svg/g[1]/g[1]/@class`
- **Port:** `class="node"`
- **Oracle:** `class="edge"`
- **Root-cause group:** G9 — outputorder=edgesfirst
- **Why deep:** DOT has `outputorder=edgesfirst`; oracle renders all edges before nodes so `g[1]`
  is an edge group (`class="edge"`), but port ignores the `outputorder` attribute and always
  renders nodes first. Porting `outputorder` requires adding a pre-pass that re-orders the
  node/edge emit loop in `emit_graph` (C: `lib/common/emit.c`).
- **Follow-on bucket:** `outputorder`

Deferred per ADR-3 (exceeds the localized ≤~30-line / single-module cutoff;
needs outputorder emit-loop reordering infrastructure). Not fixed in this mission.
