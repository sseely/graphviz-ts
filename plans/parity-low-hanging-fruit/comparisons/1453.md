# Deep case: 1453

- **Corpus path:** `1453.dot` · **engine:** `dot`
- **firstDiffPath:** `svg/g[1]/g[1]/@id`
- **Port:** `clust1`
- **Oracle:** `clust4`
- **Root-cause group:** G3 — cluster/object AGSEQ id numbering
- **Why deep:** C assigns cluster SVG ids as `clust<AGSEQ>` where AGSEQ is a global sequence
  shared across all graph objects. In 1453.dot, two anonymous rank-group subgraphs consume AGSEQ
  2 and 3 before the first named cluster, so the first cluster gets AGSEQ 4. Port uses a
  per-type counter starting at 1, producing `clust1`. Fixing requires cgraph-compatible subgraph
  sequence tracking in the Graph model.
- **Follow-on bucket:** `agseq-id-numbering`

Deferred per ADR-3 (exceeds the localized ≤~30-line / single-module cutoff;
needs AGSEQ model infrastructure across Graph/Node/Edge/Cluster). Not fixed in this mission.
