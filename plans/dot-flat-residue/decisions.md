# Architecture Decisions

## AD-1: DOT-9 routes the whole no-port no-label group through `makeSimpleFlat`

**Context:** C `make_flat_adj_edges` (dotsplines.c:1156-1166) dispatches a
no-port adjacent flat group as `labels == 0 ? makeSimpleFlat :
makeSimpleFlatLabels`. The TS no-port dispatcher only implements the
`labels > 0` half and declines otherwise, so unlabeled groups (incl.
`cnt > 1` parallel siblings) fall back to the simplified fitter and
overlap.

**Decision:** Port `makeSimpleFlat` faithfully (dotsplines.c:1075) and
route the entire no-label group through it — including the `cnt == 1`
single-edge case, exactly as C does.

**Consequences:** Easier — one faithful dispatcher matching C. Risk —
single unlabeled adjacent flats currently route via the fitter and pass
goldens; switching them could churn bytes. Mitigation: the full golden
suite is a hard gate. Goldens are generated by the C binary, which uses
`makeSimpleFlat`, so a faithful port must keep them conformant. A
churn is a porting-bug signal → STOP.

## AD-2: keep the dispatcher faithful to C's no-port branch; rename for clarity

**Context:** `makeAdjFlatLabeledEdge` is C's no-port branch but named for
only the labeled half.

**Decision:** Rename to `makeAdjFlatNoPortEdge` and update the single
caller in `edge-route.ts`. Body mirrors C: collect the no-port adjacent
group, then `labels == 0 ? makeSimpleFlat : makeSimpleFlatLabels`.

**Consequences:** Adds `edge-route.ts` to T1's write-set (one call site).
Name now matches the C control structure it ports.

## AD-3: DOT-10 reuses the aux-graph label position via the existing `updateBB`

**Context:** C copy-back (dotsplines.c:1273-1277) sets
`ED_label(e)->pos = transformf(ED_label(auxe)->pos, del, flip)`, marks it
set, and calls `updateBB(g, ED_label(e))`. The TS `copyOneFlatSpline`
copies splines + arrowheads but not the label.

**Decision:** In `copyOneFlatSpline`, when the original edge has a label,
transform the aux edge's label `pos` back and mark it set, then grow the
graph bb via the existing `updateBB` in `splines-label.ts` (export it).

**Consequences:** Relies on the rotated aux pipeline positioning the
cloned edge's label. `cloneEdge` copies the `label` *attr* string but not
the label object, so the aux graph rebuilds and positions it during
`dotInitNodeEdge` + `dotSplines_`. **Feasibility spike is T2 step 1.** If
the aux label is not positioned, STOP and reassess `cloneFlatEdge` (clone
the label object explicitly).

## AD-4: rollback / compatibility

Reversible — revert the merge commit. No data migration, no API or schema
change. Purely additive routing for previously-unhandled flat cases.
