# T2.2 — Fix (aggressively) or accept the B2 edge FP-ties

## Context
T2.1 named, per input, the deciding predicate + `file:line` and a
fixable/irreducible verdict.

## Task
For each `fixable`: apply the contraction/predicate fix (add `fma` at the
contracted `a*b+c` site; switch to the robust predicate; correct the `round()`
mode) with a unit test pinning the deciding value. For each `irreducible`:
append an accept proposal (class `A9`, disassembly/ULP evidence pointer) to
`batch-2/findings.md`. Apply verdicts to platform copies.

## Read-set
- `batch-2/findings.md` (T2.1 verdicts) · the exact sites named
- `src/common/fma.ts` · `~/.claude/rules/diagnosis.md`, `testing.md`

## Write-set
- the isolated routing/predicate src file(s) + `*.test.ts`
- `batch-2/findings.md` (append accept proposals)

## Interface input (from T2.1)
`{ input: string, verdict, predicate: string, site: "file:line",
   change?: "fma"|"robust-predicate"|"round-mode", evidence?: string,
   copies: string[] }[]`

## Acceptance criteria
- Given a fixable input, when the fix lands, then a test pins the deciding
  predicate value and tsc is clean.
- Given the fix, when a fresh scratch sfdp sweep runs, then 0 `pass->diverged`
  and the target edge id(s) move to pass.
- Given an irreducible input, when recorded, then findings.md has an A9 registry
  proposal with disassembly/ULP evidence.
- Given a fix to a shared routing primitive, when made, then the sweep confirms
  no OTHER engine's edge routing regressed (note it for Batch 6's cross-sweep).

## Observability / rollback
N/A. Reversible.

## Boundaries
- Stop if: a predicate change regresses any id, or the same predicate is
  changed 3× without clearing the tie.
- Never do: force a decision to match by special-casing an id (must be a
  general contraction/predicate fix or an accept).

## Commit
`fix(T2.2): <predicate contraction> for sfdp edge routing <input>`
