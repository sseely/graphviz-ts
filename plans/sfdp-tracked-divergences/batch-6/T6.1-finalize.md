# T6.1 — Consolidate registry, regenerate docs, full sweep

## Context
Batches 1-5 landed fixes and left accept proposals in their `batch-N/findings.md`.
This task consolidates the accepts into the registry, regenerates the parity
artifacts, and proves the whole mission with one fresh cross-engine sweep.

## Task
1. Collect every accept proposal from `batch-{1..5}/findings.md`; add each as an
   id-keyed entry (class + evidence pointer) to the per-engine accept registry
   (ADR-4). Follow the shape the accept test validates.
2. Regenerate `attribution-sfdp.jsonl` once more if any fix moved ids
   (drift-exonerated set may shift); regenerate `parity-*.{json,jsonl}` for
   sfdp AND every engine a shared-primitive fix could touch (B2/B4 flagged
   these).
3. Run a FRESH full sweep: `tsx test/corpus/parity-report.ts` after re-walking
   sfdp + any flagged engines to scratch jsonls; verify 0 `pass->diverged` on
   every engine and that the sfdp tracked count dropped.
4. Regenerate `PARITY.md` + `PARITY-<engine>.md`.
5. Append a mission-summary row to `plans/decision-journal.md` (root repo) AND
   this mission's `decision-journal.md`: tracked before/after, fixes landed,
   accepts added (with classes), sweep result.

## Read-set
- `batch-{1..5}/findings.md` (accept proposals + which engines to re-sweep)
- `test/corpus/accepted.ts`, `test/corpus/accepted-divergences-engines.test.ts`
  (registry shape + the validating test)
- `test/corpus/parity-report.ts`, `test/corpus/engine-walk.ts`

## Write-set
- the accept-registry file (e.g. `test/corpus/accepted.ts` or the per-engine
  accept source) — SOLE writer
- `test/corpus/parity-*.json`, `test/corpus/parity-*.jsonl` (re-swept engines)
- `test/corpus/PARITY*.md`, `test/corpus/attribution-sfdp.jsonl`
- `plans/sfdp-tracked-divergences/decision-journal.md`, `plans/decision-journal.md`

## Acceptance criteria
- Given all accept proposals, when added, then
  `accepted-divergences-engines.test.ts` passes and every accepted id has an
  evidence pointer (ADR-2).
- Given the full fresh sweep, when diffed vs baselines, then 0 `pass->diverged`
  on every engine (sfdp, neato, fdp, circo, twopi, osage, patchwork).
- Given the mission complete, when PARITY-sfdp.md is read, then tracked count is
  strictly lower than 50 and equals (diverged − drift-exonerated − new-accepts).
- Given `npx tsc --noEmit` and `npm test`, when run, then both are clean/green.

## Observability / rollback
N/A. Reversible.

## Boundaries
- Stop if the full sweep shows ANY `pass->diverged` regression on any engine —
  bisect to the offending bucket's fix before finalizing.
- `pgrep` before every sweep; scratch jsonls only; never resume.
