# T4.1 — Analyze the B4 ratio=fill residuals

## Context
`ratio=fill` layouts are post-scaled by `_neato_set_aspect` fill/expand factors
computed from GD_bb and the target size. The scale multiplies all coordinates,
amplifying any pre-scale difference. Two possible roots: (a) the scale FACTOR
differs from native (fixable), (b) the scale is correct but pre-scale positions
drift (accept — amplified drift).

## Task
For each B4 rep (trapeziumlr, pgram, 1855): inject exact PRE-scale native
positions, re-run the port's aspect scaling, and compare (i) the computed scale
factor and (ii) the final scaled bb to native. Attribute the residual to
scale-math vs amplified-drift per `diagnosis.md`. If pgram still can't be
injected (harness-error), instrument the injector for its input shape and note
the blocker.
