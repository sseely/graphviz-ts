# T1.1 — Analyze the graph-bb residual (representative + each distinct input)

## Context
sfdp graph-bb divergences: firstDiff is `[graph] _draw_ filled_polygon` — the
graph bounding box side differs by a few points. The bb is derived from the max
node/edge extent, so a bb diff is downstream of a position, label-box, or
spline-hull difference. graphviz-ts stores GD_bb per (sub)graph; sfdp's final bb
is the edge CURVE (no post-routing compute_bb) — see
[[sfdp-graphbb-hull-vs-curve-entangled]].

## Task
For the B1 representative (graphs-unix) and each other distinct input T0.2
assigned to B1: inject exact native positions and determine whether the bb
still diverges. State the mechanism per `diagnosis.md` (cause, `file:line`,
causal chain, ruled-out) BEFORE proposing any fix. Record per-input verdict:
`fixable:<mechanism>` or `irreducible:<controlled-experiment>`.

## Method (per diagnosis.md — non-negotiable)
1. `GVTS_POS_DUMP=1 dot -Ksfdp -Txdot <input>` → full-precision positions.
2. `GVTS_POS_INJECT=<dump> render-one-xdot <input> sfdp`; compare bb to native.
3. If bb now matches → id is drift (recheck why T0.1 marked it not-cleared).
4. If bb still diverges → instrument which extent sets the divergent side
   (node label box width/height, edge curve hull, xlabel bb) and trace to the
   originating `file:line`. Rule out alternatives with evidence.

## Acceptance criteria
- Given each B1 input, when injected + traced, then findings.md records a
  one-sentence mechanism + `file:line` + a fixable/irreducible verdict.
- Given an "irreducible" verdict, when written, then it cites a controlled
  experiment (not an assertion) isolating the FP variable.
- Given a "fixable" verdict, when written, then it names the exact change site
  for T1.2.
