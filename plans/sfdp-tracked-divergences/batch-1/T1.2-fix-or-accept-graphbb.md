# T1.2 — Fix (aggressively) or accept the B1 graph-bb residuals

## Context
T1.1 classified each B1 input `fixable:<mechanism@file:line>` or
`irreducible:<experiment>`. Apply the verdict.

## Task
For each `fixable` input: implement the minimal fix at the mechanism origin
(diagnosis.md — fix the cause, not the symptom); add/extend a unit test that
would fail without it. For each `irreducible` input: append a proposed
accept-registry entry to `batch-1/findings.md` (class `A-bb`, evidence pointer)
— do NOT edit the registry (Batch 6 owns it). Apply every verdict to the
input's platform copies (same input → same mechanism).

## Read-set
- `batch-1/findings.md` (T1.1 verdicts)
- the specific `file:line` sites named by T1.1 (load only those ranges)
- `~/.claude/rules/diagnosis.md`, `~/.claude/rules/testing.md`

## Write-set
- the isolated src file(s) named in T1.1 (one writer; if two B1 inputs need the
  same file, do them in one commit) + their `*.test.ts`
- `batch-1/findings.md` (append accept proposals)

## Interface input (from T1.1)
`{ input: string, verdict: "fixable"|"irreducible", site?: "file:line",
   experiment?: string, copies: string[] }[]`

## Acceptance criteria
- Given a fixable input, when the fix lands, then a new/extended test fails on
  the pre-fix code and passes after, and `npx tsc --noEmit` is clean.
- Given the fix, when a FRESH sfdp engine-walk to a scratch jsonl runs, then 0
  `pass->diverged` regressions vs baseline and the target id(s) move to pass.
- Given an irreducible input, when recorded, then findings.md has a registry
  proposal with a controlled-experiment evidence pointer (no bare assertion).
- Given a representative verdict, when applied, then every platform copy is
  re-swept and shares the outcome.

## Observability / rollback
N/A. Reversible (revert the commit).

## Boundaries
- Stop if: the same site is changed 3× without clearing the diff
  ([[bucket-fix-rebucketing]]); a fix regresses any id.
- Never do: edit the accept-registry file (Batch 6 only); widen a fix to a
  shared primitive without stating why it must be broad.

## Commit
`fix(T1.2): <mechanism> for sfdp graph-bb <input-family>` (one per fixable
family) · accept proposals carried in findings.md, committed by Batch 6.
