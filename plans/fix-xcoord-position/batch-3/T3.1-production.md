# T3.1 — Production wiring + docs

## Context
Finalize the production defaults (ADR-5/6) and document the measurement contract:
deterministic for rules/tests, host-faithful for production.

## Task
1. Confirm production defaults via the T1.1 resolution chain:
   - browser → browser `CanvasTextMeasurer`
   - Node → node-canvas system measurer (lazy) when present
   - Node, no canvas → `EstimateTextMeasurer` + install-advice warning (ADR-5)
2. Add a node-canvas-backed system measurer for Node if not already present in
   T1.1 (wrap node-canvas's `measureText`; map font names like the browser path).
3. **Docs:** a short `docs/` note (or README section) stating the contract:
   - rules/tests are deterministic (reference estimate vs headless dot);
   - production measures with the host font (kerning/ligatures), so output may
     differ across hosts — by design, matching graphviz's own behavior;
   - how to override via `setTextMeasurer`; how to install `canvas` for Node.
4. Ensure the `package.json` declares `canvas` as **optional** (optionalDependency
   or peerDependency with `peerDependenciesMeta.optional`), not a hard runtime dep.

## Read-set
- `src/common/textmeasure-factory.ts` (resolution chain from T1.1)
- `package.json`
- DESIGN.md §4.3, §6, ADR-5/6

## Acceptance criteria (Given/When/Then)
- Given Node with `canvas` installed and no override, when laid out, then the
  node-canvas system measurer is used (host font, kerned).
- Given Node without `canvas`, then `EstimateTextMeasurer` + one-time install
  advice; no crash.
- Given a browser build, then no `canvas` in the bundle; browser canvas used.
- Given the docs, then the deterministic-vs-host-faithful contract is stated and
  `setTextMeasurer` + canvas-install are documented.
- `npm test`, `npm run typecheck`, lizard → pass/clean.

## Quality bar
Commit `feat(textmeasure): host-faithful Node production + measurement docs`.
