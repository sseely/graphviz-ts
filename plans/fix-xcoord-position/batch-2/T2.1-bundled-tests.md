# T2.1 — Bundled-font measurement tests (kerning, ligatures, charset)

## Context
Font fidelity (the layer decoupled from rules) needs its own reproducible,
cross-platform tests. Reading the *system* font would be platform-dependent;
instead conjure metric tables from **committed** fonts via a pure-JS shaper, so
the tables are identical on Mac/Windows/Linux (DESIGN.md §5.2).

## Task
1. **Bundle reference fonts** under `test/fonts/` — open-license, metric-compatible
   with the corpus families: e.g. Liberation Serif/Sans/Mono (Times/Helvetica/
   Courier), plus coverage for Symbol and a Latin1 case. Record provenance +
   license. (Push-forward: pick the set, log it.)
2. **Conjure tables at test time** from the bundled fonts with a pure-JS reader:
   - `fontkit` (or `opentype.js`) → per-glyph advances + kern/GPOS pairs.
   - Add `harfbuzzjs` ONLY if ligature coverage (`<=`, `!=`, `=>`) needs full
     shaping beyond pair kerning.
3. **Unit tests** asserting the conjured data is correct against the font's own
   tables:
   - kerning: a kern pair (e.g. `WA`) has the expected negative adjustment;
   - ligatures (if a ligature font is bundled): `<=` shapes to one cluster /
     single advance vs two glyphs in a non-ligature font;
   - charset: non-ASCII (Latin1) and Symbol-font code points resolve to the
     font's advances (covers the `Latin1`/`Symbol` corpus non-matches).
4. These tests validate the *measurement* layer in isolation — they do NOT touch
   the layout corpus.

## Read-set
- DESIGN.md §5.2, §2 (Latin1/Symbol are the charset non-matches)
- `src/common/textmeasure.ts` (interface the conjured measurer implements)
- fontkit / harfbuzzjs docs (dev deps)

## Acceptance criteria (Given/When/Then)
- Given a bundled font + JS shaper, when tables are conjured, then they are
  identical across platforms (derived only from committed inputs).
- Given a kern pair, when measured, then the kern adjustment matches the font's
  kern/GPOS value.
- Given a ligature font (if bundled) and `<=`, then it shapes to a single
  advance; given a non-ligature font, then two.
- Given Latin1 / Symbol code points, then advances match the font (no silent
  space-width fallback for covered glyphs).
- Given `npm test`, `npm run typecheck`, lizard → pass/clean. Existing survey
  still 0 regressions.

## Boundaries
- Bundled fonts are **test/dev-only** assets; do not ship them in the runtime
  package. JS shaper deps are devDependencies.

## Quality bar
Commit `test(fonts): bundled-font measurement tests (kern/ligature/charset)`.
