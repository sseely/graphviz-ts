# T0.2 — Headless rules corpus (side-by-side)

## Context
Validate that the port's layout **rules** match graphviz by comparing the port
(with `EstimateTextMeasurer`) against native dot run **headless** (no textlayout
plugin → `estimate_textspan_size`). This stands up **alongside** the existing
pango-goldens survey (ADR-3); the existing survey stays green until Batch 3.

## Task
1. **Headless oracle generation** (validated recipe):
   ```
   mkdir /tmp/ghl
   cp <build>/.../libgvplugin_core.* libgvplugin_dot_layout.* /tmp/ghl/
   GVBINDIR=/tmp/ghl dot -c                 # regen config; textlayout empty
   GVBINDIR=/tmp/ghl dot -Tsvg g.gv         # → estimate (verify: -v shows
                                            #   "resolved to: [internal hard-coded]")
   ```
   Script this as a corpus-golden generator (e.g. `test/corpus/gen-headless.ts`)
   that writes headless goldens to a new dir (e.g. `test/corpus/golden-headless/`).
2. **Rules survey** (`test/corpus/survey-rules.ts` or a flag on survey.ts): render
   each corpus graph with the port forced to `EstimateTextMeasurer`
   (`setTextMeasurer` from T1.1, or inject directly for now) and compare to the
   headless goldens. Report per-graph max node-position delta; deterministic
   tolerance (≤ 0.5pt to count as match, but expect 0.00).
3. **Isolate pre-existing divergences:** any graph that ALSO diverges under the
   current default (LUT vs pango) for a font-independent reason (e.g. `Petersen`,
   `Latin1`/`Symbol` charset) is recorded in a `rules-known-divergences.md`
   allowlist with the reason; it does not fail the gate.
4. Record the pass/fail tally in `decision-journal.md`.

## Read-set
- `test/corpus/survey.ts`, `test/corpus/render-one.ts` (existing harness shape)
- DESIGN.md §5.1 (recipe), §2 (spike numbers, the 4 known non-matches)
- repro/b69-min-noconc.gv

## Acceptance criteria (Given/When/Then)
- Given the headless generator, when run, then `dot -v` confirms the estimate path
  (`[internal hard-coded]`) and goldens are produced for the corpus.
- Given the rules survey (port `EstimateTextMeasurer` vs headless goldens), when
  run, then **every graph is byte-exact (max delta 0.00)** except entries in
  `rules-known-divergences.md`, each with a font-independent reason.
- Given b69-min-noconc, then max node delta == 0.00.
- Given the EXISTING survey (LUT vs pango goldens), when run, then **0 regressions**
  vs `/tmp/parity.before.json` (side-by-side invariant, ADR-3).

## Observability / rollback
New files only (generator, rules survey, headless goldens, allowlist). Reversible.

## Quality bar
Commit `test(corpus): add side-by-side headless rules survey`. The rules survey
is additive; the existing survey is untouched.
