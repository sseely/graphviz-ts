# T0.1 — Add EstimateTextMeasurer (raw, headless-matching)

## Context
The reference measurer for the rules corpus must reproduce graphviz's
`estimate_textspan_size` exactly: raw per-char widths (no hinting), no kerning,
height = `fontsize * LINESPACING(1.20)`. The port already has the width function
(`estimate_text_width_1pt` in `src/common/textmeasure.ts`); it just needs to be
exposed as a `TextMeasurer`. Spike-proven to byte-match headless dot (ADR-1).

## Task
1. Add `EstimateTextMeasurer implements TextMeasurer` in `src/common/textmeasure.ts`:
   - `w = fontsize * estimate_text_width_1pt(fontname, text, bold, italic)`
   - `h = fontsize * 1.20`  (graphviz `LINESPACING`; add a named constant)
   - Mirror the bold/italic flag handling of the other measurers.
2. Export it. Do NOT change `LutTextMeasurer` or `CanvasTextMeasurer` yet (ADR-4
   demotion happens in T1.1).
3. Unit test (`textmeasure.test.ts` or sibling): assert raw widths for a few
   strings equal `fontsize * estimate_text_width_1pt(...)` and height = `1.2*size`
   for Times, Helvetica, Courier; assert NO per-char px rounding occurs
   (e.g. a string whose hinted width differs from raw width yields the raw value).

## Read-set
- `src/common/textmeasure.ts` (`estimate_text_width_1pt`, `freetypeHintedWidth`,
  `LutTextMeasurer`, `CanvasTextMeasurer`, `TextMeasurer`)
- `~/git/graphviz/lib/common/textspan.c:estimate_textspan_size` (the C spec:
  `size.x = fontsize * estimate_text_width_1pt`, `size.y = fontsize * LINESPACING`)
- DESIGN.md §4.2, §5.1

## Acceptance criteria (Given/When/Then)
- Given a kern/hint-sensitive string (e.g. "WAR-WR1VI1"), when measured by
  `EstimateTextMeasurer` at 14pt, then `w == 14 * estimate_text_width_1pt(...)`
  (raw; NOT the hinted value `freetypeHintedWidth`).
- Given any string, when measured, then `h == fontsize * 1.20`.
- Given `npm test` and `npm run typecheck`, then both pass.
- Given lizard on the edited file, then clean (-C 10).
