# T2 — New golden tests (~12–15)

## Context

The golden suite has 50 entries covering 8 engines but low graph-feature
diversity (no rankdir variants, self-loops, multi-edges, single-node
graphs, minlen/constraint attrs). Refs are SVGs generated by the C
graphviz 15.0.0 binary installed on this machine (`dot -V` to confirm
15.0.0 — STOP if it isn't). decisions.md D2/D6/D7 govern provenance,
engine bias, and quarantine.

## Task

1. Write new .dot inputs in test/golden/inputs/, named
   `<engine>-<feature>.dot`. Candidate list (pick ~12–15; deterministic
   engines first per D6):
   - dot: rankdir-lr, rankdir-bt, self-loop (`A->A`), multi-edge
     (`A->B; A->B; A->B`), minlen (`A->B[minlen=2]`), constraint-false,
     head-tail-label (`A->B[headlabel="h" taillabel="t"]`)
   - twopi: single-node, self-loop
   - circo: single-node, self-loop
   - osage: deep-nesting (4 levels)
   - patchwork: single-node
   - neato: tiny-multi-edge (3 nodes, one doubled edge)
   - fdp: tiny-self-loop (3 nodes + `A--A`)
   - sfdp: single-node
2. Generate each ref: `dot -K<engine> -Tsvg test/golden/inputs/X.dot
   > test/golden/refs/X.svg`.
3. For each: run the port (vitest or a probe) and compare.
   - PASSES at its engine's tolerance class → append manifest entry
     (description records: "ref: graphviz 15.0.0 dot -K<engine> -Tsvg").
   - Iterative engine within ~10pt but over 0.5pt → tolerance +
     portReference pin per D2 (journal entry; copy the M8 pattern in
     manifest.json sfdp-medium/-large).
   - FAILS structurally or beyond pinning → QUARANTINE per D7: move
     input to test/golden/quarantine/, keep the ref beside it, journal
     entry, do NOT add to manifest.
4. Update suite.test.ts `manifest has 50 entries` assertion to the new
   count; update run.sh's hardcoded "50 passed" check to derive the
   count from the manifest (jq length).
5. STOP CONDITION: if more than half the candidates quarantine, stop
   and report (systemic gap).
