# T0.3 — Re-bucket the not-cleared set + refined bucket list

## Context
After T0.2, `attribution-fdp.jsonl` holds the fresh drift-exonerated /
not-cleared split. The not-cleared ids are the real tracked residual; group them
into the mission's buckets so Batches 1-3 target real work (and skip buckets
emptied by the regen).

## Task
For each not-cleared id compute: (a) injection signature (what survives exact
positions — the discriminator, NOT firstDiff which is often the graph bg
polygon), (b) `injectedDiffs`/`baseDiffs`, (c) input attributes (edge labels /
lp, ratio, size, empty-named nodes), (d) connected-component count (single vs
multi — like the sfdp mission, this decides which postprocess path applies).
Emit `batch-0/findings.md`: one row per DISTINCT input, its platform copies, its
bucket, and a one-line mechanism hypothesis. Flag any bucket now empty.

Buckets:
- **B1 frame/postprocess** — residual signature is edge-label (`lp`/`_ldraw_`)
  or a constant frame offset (dx=0, const dy). Candidates: graphs-cairo, 2193.
- **B2 force-drift** — injection CLEARS it (should be drift-exonerated; if any
  unix-family id is still not-cleared, investigate why — real residual vs a
  no-op injection on a space-named-node graph).
- **B3 FP-tie (A9)** — segment-count flip (opCount/ptCount) or sub-pixel
  arrowhead drift. 241_0, 2095 (shared with sfdp).

## Interface output (consumed by Batches 1-3)
```
findings.md table columns:
  representative: string
  copies: string[]            (platform-duplicate ids sharing the input)
  bucket: "B1" | "B2" | "B3"
  components: number          (single vs multi — postprocess path)
  signature: string           (injection residual kinds)
  attrs: string[]             (edge-label | ratio | size | empty-name | plain)
  hypothesis: string
```

## Acceptance criteria
- Given the not-cleared set, when bucketed, then every id appears in exactly one
  representative row (as representative or copy).
- Given a bucket with no surviving not-cleared ids, when writing findings, then
  it is explicitly marked EMPTY (its batch is skipped).
- Given findings.md, when read by a later batch, then the representative id,
  bucket, and component-count are unambiguous.
