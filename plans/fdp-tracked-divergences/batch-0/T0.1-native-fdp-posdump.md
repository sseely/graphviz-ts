# T0.1 — Restore the native fdp GVTS_POS_DUMP + rebuild

## Context
The injection-attribution harness needs the native oracle to dump fdp
pre-routing `ND_pos` on stderr (`GVTS_POS <name> <x> <y>`). Today the dump only
exists in `spline_edges` (`lib/neatogen/neatosplines.c:841`), which fdp does NOT
call — fdp routes through `spline_edges1`, so `GVTS_POS_DUMP=1 dot -Kfdp` emits 0
lines. The port already injects at the correct site
(`src/layout/fdp/index.ts:91`, before `neatoSetAspect`/routing), and its comment
names the native dump site: `fdp_layout`, between `fdpLayout()` and
`neato_set_aspect()` (`lib/fdpgen/layout.c` ~:1062).

2. Investigate (ADR-1) whether any existing hook covers fdp before adding code —
   grep `lib/fdpgen/*.c` for an existing/commented dump; if none, proceed.

   the node-position finalization (`fdpLayout()` returns) and BEFORE
