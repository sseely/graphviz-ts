# Architecture decisions — structural-match buckets

Confirmed with the user (all three recommended options).

## AD-1: Cluster signal = worst-diff element-kind × magnitude band

- **Context.** Structural-match rows carry only `maxDelta` (a scalar); there is
  no location signal to cluster on.
- **Decision.** The auto-dashboard buckets by the **element kind of the worst
  numeric diff** (`edge-path @d` / `node-ellipse @cx,@cy` / `polygon @points` /
  `text @x,@y,@transform` / `other`) crossed with a **coarse magnitude band**
  (`1–10` / `10–100` / `100–1000` / `>1000`; there are 0 sub-pixel cases).
- **Consequences.** Deterministic and regenerates on every survey. Coarser than
  semantic edge/node/cluster attribution — that richer attribution lives in the
  Batch 4 analysis doc (point-in-time, human/agent-verified), not the tooling.

## AD-2: Location signal is captured in the survey (durable), not a one-off

- **Context.** The user wants the section "generated by dashboard.ts", which
  requires the signal in `parity.json`.
- **Decision.** Add an **additive** `maxDeltaPath?: string` field to
  `SurveyResult`, set by `diffVerdict` to the `path` of the worst numeric diff.
  A full re-survey populates it; the dashboard section then regenerates forever.
- **Consequences.** Costs one full survey run. The field is inert to
  `rules-gate.ts` (reads only verdict/maxDelta/firstDiffPath/clipOverflow), so it
  cannot cause a gate regression on its own.

## AD-3: Do NOT perturb the golden comparator (`test/golden/compare.ts`)

- **Context.** Semantic edge/node/cluster buckets would want the SVG element's
  `class` in the diff path.
- **Decision.** Leave `compare.ts` untouched. `maxDeltaPath` is the raw XPath the
  comparator already emits. Semantic class attribution is done in Batch 4 by
  inspecting the cached SVGs, not by changing shared diff paths.
- **Consequences.** Zero risk to the golden snapshot suite. Auto-buckets stay
  element-kind coarse (AD-1); semantic depth is analysis-only.

## AD-4: `maxDeltaPath` records the path of the MAX numeric diff, deterministically

- **Context.** Multiple numeric diffs can tie on delta.
- **Decision.** Record the path of the diff with the greatest `delta`; on ties,
  the **first-encountered** (compareSvg walk order) wins — a stable, order-defined
  tie-break. Store `undefined` only when there are no numeric diffs (cannot happen
  for structural-match, which is defined by ≥1 numeric diff and 0 structural).
- **Consequences.** One reduce pass; no new traversal. Faithful to the existing
  `maxDelta` reduce (survey.ts:322) — same source array, same order.

## AD-5: Analysis attributes to KNOWN families first; novelty is the exception

- **Context.** The corpus already has an extensive diagnosed-mechanism record in
  `.agent-notes/` + auto-memory. Most of the 163 are residuals of known families.
- **Decision.** Batch 4 agents must **cross-reference the seed catalog
  (README §Known mechanism families) and `.agent-notes/` before proposing a novel
  cause.** A case is "novel" only after the known families are ruled out with a
  one-line reason. Follows the global diagnosis rule (instrument/attribute before
  hypothesizing).
- **Consequences.** Turns 163 scattered near-misses into ~6–10 fixable classes
  anchored to prior evidence, not fresh guesses.

## Stop conditions {#stop-conditions}

**STOP and wait for human input when:**
- `survey:gate` (`rules-gate.ts`) reports **any** regression after the
  re-survey. The tooling change must be verdict-inert; a regression means it was
  not. Do **not** `cp parity-rules.json parity.json` until gate = 0.
- The `maxDeltaPath` addition changes any existing `verdict` or `maxDelta` value
  (diff `parity.json` numeric/verdict columns before/after — must be additive).
- A task needs to modify a file outside its declared write-set.
- Two consecutive quality-gate failures on the same check.
- The re-survey's structural-match count drifts materially from 163 (the oracle
  cache went stale or the measurer/GVBINDIR was wrong) — verify recipe before
  proceeding.

**PUSH FORWARD with judgment when:**
- Naming buckets / magnitude-band boundaries (stylistic; AD-1 sets defaults).
- Analysis-doc prose, ordering, and cross-reference wording.
- A structural-match case is trivially a known family with a matching
  `.agent-notes/` entry — cite it and move on.

## Rollback

Reversible. `git revert` the tooling commits, then
`npx tsx test/corpus/dashboard.ts` regenerates `PARITY.md` from the restored
`parity.json`. No data migration, no external state.
