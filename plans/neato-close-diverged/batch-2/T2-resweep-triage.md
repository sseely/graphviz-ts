# T2 · Fresh re-sweep + residual triage

## Context

After T1 lands, node positions have moved toward the oracle. The pre-B1 bucket
lists in `buckets.json` are now stale — spline/label/arrow items that were pure
cascades of a mislaid node will have flipped to `pass`. This task produces the
true residual, which is Batch 3's input.

## Task

1. Confirm T1 is committed and the tree is clean.
2. Run a **fresh** (deleted-JSONL) full neato sweep:
   `rm -f test/corpus/parity-neato.jsonl && GVBINDIR=/tmp/ghl npx tsx
   test/corpus/engine-walk.ts neato` → regenerates `test/corpus/parity-neato.json`.
3. Re-triage every remaining `diverged` row into a bucket using the same
   signature logic used to build `buckets.json`:
   - B2 = `edge:… _draw_ …unfilled_bezier` · B3 = `cluster:… _draw_` ·
     B4 = `edge:… _ldraw_` · B5 = `edge:… _h(l)draw_`.
4. For each residual id, record: `{ id, bucket, firstDiff, nDiffs,
   suspectedCause }` where `suspectedCause ∈ {genuine, cascade-suspect}` —
   flag any whose firstDiff object references a node whose position T1 changed as
   `cascade-suspect` (Batch 3 verifies).
5. Write `plans/neato-close-diverged/residual-tracker.md` (the per-id table).
   No source edits.

## Write-set

  `parity-neato.json` is regenerated and its diverged count ≤ 51 (the pre-B1
  non-graph-fill total) — the drop from 95 is the B1 win + cascade collapse.
- **Given** the residual list, **when** triaged, **then** every diverged id is
  assigned exactly one of B2/B3/B4/B5 and a `suspectedCause`.
- **Given** the gate, **then** `bash test/golden/gates.sh` still exits 0 (sanity;
  no src changed).
