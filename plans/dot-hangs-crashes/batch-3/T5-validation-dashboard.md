# T5 — Validation + dashboard

## Context

This task proves the mission met its bar and updates the parity artifacts.
Read `decisions.md#ad-4` and the project CLAUDE.md completion rule: a batch with
any quarantined/excluded case is not complete until its comparison page exists
and is referenced in the decision journal.

## Task

1. **Capture baseline** (do this at mission start, before batch-1, if not already
   saved): copy `test/corpus/parity.json` to `/tmp/parity-baseline.json` and note
   the conformant / structural-match counts.
2. Regenerate the survey and dashboard:
   `npx tsx test/corpus/survey.ts && npx tsx test/corpus/dashboard.ts`.
3. **Regression check (hard gate):** diff new `parity.json` per-id verdicts vs
   baseline. **No** id may move from `conformant`→worse or `structural-match`→
   `diverged`/`errored`/`timeout`. If any does → STOP, revert the offending
   change, log it.
4. **Verify rescues:** the 7 former `timeout` ids (`1718`, `2108`, `2222`,
   `2471`, `2475_2`, `graphs-b100`, `graphs-b104`) must no longer be `timeout`.
   Record each one's new verdict and its port-vs-native timing.
5. **3×-native check:** for each rescued case, confirm port time ≤ ~3× native
   (hard floor: < 20s). Log the table. Any case missing 3× after T1+T3 is a
   documented residual, not a failure — note it explicitly.
6. **Comparison pages:** for every rescued case that lands in `diverged`, write a
   comparison page under `comparisons/` (follow the existing page format in that
   dir) and reference it in `decision-journal.md`.
7. **Parser outcomes:** confirm the 5 `errored` ids reflect T4's outcome (fixed →
   gone from errored; won't-fix → still errored with a journal entry).
8. Append the final mission summary to `README.md` (tasks done, regressions=0,
   final timings, residuals, won't-fix list).

## Acceptance criteria

- **Given** baseline vs post-mission `parity.json`, **when** diffed, **then**
  conformant and structural-match counts are **≥** baseline (zero regressions).
- **Given** the 7 former timeouts, **when** the survey re-runs, **then** none is
  `timeout`; each has a logged new verdict + timing.
- **Given** each rescued `diverged` case, **when** the mission closes, **then** a
  comparison page exists and is referenced in the journal.
- **Given** `npm test` + `npm run typecheck`, **when** run on the final branch,
  **then** both exit 0.
